## Key Recent Facts

- GyanBrige is a live-class edtech platform: students attend real-time lectures, audio is transcribed (Whisper), AI generates structured notes.
- **Web**: Next.js 16, TypeScript, Tailwind v4. App Router. Pages: login, signup, dashboard, courses.
- **API routes** (web/src/app/api/): auth, courses, lectures, transcribe, generate-notes, livekit, livestreams, attendance, progress, enrollments, upload, users.
- **Backend**: standalone Node.js/Express microservice at `backend/transcriptionServer.js`. Uses OpenAI SDK v4. File upload via multer. Notes generated by `services/noteGenerator.js`.
- **Mobile**: React Native at `GyanBrige/`. Screens, navigation, services, theme directories.
- **Streaming**: LiveKit (livekit-client v2.17 + livekit-server-sdk v2.15) for real-time class sessions.
- **Transcription**: OpenAI Whisper via `/transcribe` route and backend service. Uploads stored in `backend/uploads/`.

## Known Architecture Notes

- The backend transcription service is a separate process from Next.js (not a Next.js API route). Next.js calls it via HTTP.
- `generate-notes` and `transcribe` are both Next.js API routes AND backend services — check which layer a given route uses before editing.
- LiveKit token generation happens server-side in the `livekit` API route.
